          # React简介

          • 官方网站（中文） (opens new window)
          • 官方网站（英文） (opens new window)

          # 1.1 介绍描述

          • 用于动态构建用户界面的 JavaScript 库(只关注于视图)
          • 由Facebook开源

          # 1.2 React特点

          • 采用==组件化模式、声明式编码==，提高开发效率及组件复用率
          • 在==React Native==中可以使用React语法进行==移动端开发==
          • 使用==虚拟DOM+优秀的Diffing算法==,尽量减少与真实DOM的交互

          # 1.3 React为何高效？

          • 使用虚拟(virtual)DOM, 不总是直接操作页面真实DOM
          • DOM Diffing算法, 最小化页面重绘

          # 1.4 初次使用

          # 1.4.1 相关JS库

          • react.js：React核心库
          • react-dom.js：提供操作DOM的react扩展库
          • babel.min.js：解析JSX语法代码转为JS代码的库

          # 1.4.2 创建虚拟DOM

          • 纯JS方式(一般不用)
          • JSX方式
          <script type="text/babel" > /* 此处一定要写babel */
  //1. 创建虚拟DOM 此处一定不要写引号，因为不是字符串
  const VDOM = <h1>Hello,React</h1>
  //2. 渲染虚拟DOM到页面
  ReactDOM.render(VDOM,document.getElementById('root'))
</script>

          # 1.4.3 虚拟DOM和真实DOM

          • 虚拟DOM本质就是Object类型
          • 虚拟DOM本身属性比较少（只需在React内部使用）
          • 真实DOM本身属性比较多
          • 虚拟DOM最终会被转换为真实DOM

          # 1.5 JSX语法

          • 全称： JavaScript XML
          • 是React定义的一种类似于XML的JS扩展语法：JS + XML。
          • 作用：用来简化创建虚拟DOM
            • 写法：let ele = <h1>Hello JSX!</h1>
            • 它不是字符串, 也不是HTML/XML标签
            • 它最终产生的就是一个JS对象
          • 标签名任意：HTML标签或其它标签
          • 标签属性任意：HTML标签属性或其它
          • 基本语法规则
            • 对于要使用变量名的情况需要用{username}包裹变量名
            • 对于样式的使用要使用className="container"
            • 对于内联样式需要使用双重花括号包裹
            • 不能有多个根标签，只允许设置一个
            • 标签必须闭合
            • 标签首字母
              • 若小写字母开头，则将该标签转为html中同名元素，若html中无该标签对应的同名元素，则报错
              • 若大写字母开头，React就去渲染对应的组件，若组件没有定义，则报错
          • babel.js的作用
            • 浏览器不能直接解析JSX代码, 需要babel转译为纯JS的代码才能运行
            • 只要用了JSX，都要加上type="text/babel", 声明需要babel来处理
